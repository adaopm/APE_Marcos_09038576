Git - principais comandos utilizados no Git e suas finalidades

O Git é um sistema de controle de versão amplamente utilizado para rastrear mudanças em projetos de software e colaborar em equipe. Existem muitos comandos do Git disponíveis, cada um com sua finalidade específica.
Aqui você verá alguns dos principais comandos do Git e suas finalidades:

Estes são apenas alguns dos comandos Git mais comuns, e que sua utilização depende do objetivo aque se deseja alcançarcomo desenvolviemnto do projeto. Portanto, é recomendável aprender mais sobre o Git e suas funcionalidades para usá-lo de forma eficiente.


git init : Inicializa um repositório Git em um diretório local, criando um novo repositório vazio ou reutilizando um existente.

git clone: Cria uma cópia local de um repositório Git remoto em seu computador.

git add: Adiciona arquivos ao índice (staging area), preparando-os para serem commitados.

git commit: Registra as mudanças no repositório localmente com uma mensagem que descreve as alterações realizadas.

git status: Exibe o estado atual do repositório, mostrando quais arquivos foram modificados, adicionados ou removidos.

git log: Mostra o histórico de commits, incluindo informações como autor, data e mensagem de cada commit.

git branch: Lista todas as ramificações (branches) disponíveis no repositório.

git checkout: Permite alternar entre diferentes ramificações ou commits.

git merge: Combina mudanças de uma ramificação em outra, geralmente usada para incorporar o trabalho de uma ramificação em outra (merge).

git pull: Atualiza seu repositório local com as mudanças do repositório remoto.

git push: Envia as mudanças locais para um repositório remoto.

git remote: Exibe informações sobre repositórios remotos vinculados ao seu projeto.

git fetch: Recupera informações sobre as mudanças em um repositório remoto, mas não as aplica localmente.

git diff: Mostra as diferenças entre arquivos ou commits.

git reset: Permite desfazer commits, reverter mudanças no índice ou reverter mudanças em arquivos específicos.

git stash: Permite guardar temporariamente as mudanças não commitadas para trabalhar em outra tarefa e, em seguida, recuperá-las.

git tag: Permite adicionar tags a commits específicos para marcar versões ou momentos importantes.

git remote add: Adiciona um novo repositório remoto.

git remote remove: Remove um repositório remoto.

git rm: Remove arquivos do repositório e os marca para serem excluídos no próximo commit.
