Git - principais comandos utilizados no GitFlow e suas finalidades

O Git Flow é um modelo de fluxo de trabalho muito utilizado por equipes de que trabalham de modo colaborativo no desenvolvimento software, se destacando por auxiliar na organização do versionamento de códigos.

Importante ressaltar que para utilização do GitFlow em ambiente Windows é necesário configurar seu computador.
-- Realizar a instação do CygWin .
-- Realizar a instação do Wget .
-- Importante : o arquivo wget.exe deve ser colado dentro da pasta system32

Intalando o GitFlow:
É necessário que o git esteja instalado como pré-requisito.
$ wget -q -O - --no-check-certificate https://raw.github.com/petervanderdoes/gitflow-avh/develop/contrib/gitflow-installer.sh install stable | bash

Versionamento com Git flow



git flow release: cria um branch da versão baseado no branch 'develop'.

git flow release start RELEASE [BASE]: o desenvolvedor, caso queira, pode informar um hash sha-1 do commit [BASE] de onde começar a versão, porém o commit precisa estar no branch 'develop'.

git flow release publish RELEASE: publicar o branch da versão após criá-lo, para permitir commits por outros desenvolvedores.

Ao realizar a finalização da versão são executados alguns passos:
º Mescla o branch da versão no 'master'
º Etiqueta a versão com seu nome
º Mescla o branch da versão de volta no 'develop'
º Remove o branch da versão
º git flow release finish RELEASE


Os subcomandos disponíveis são:
init Inicializa um novo repositório git com suporte para o modelo de ramificação.
feature Gerencie suas ramificações de recursos.
bugfix Gerencie seus ramos de correção de bugs.
release Gerencie suas ramificações de lançamento.
hotfix Gerencie suas ramificações de hotfix.
support Gerencie suas filiais de suporte.
version Mostra informações sobre a versão.
config Gerencie sua configuração do git-flow.
log Mostra o log desviando do branch base.
